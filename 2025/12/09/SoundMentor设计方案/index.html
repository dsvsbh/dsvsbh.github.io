一、 背景

当前，教育数字化转型已进入深水区，教学模式正从“以教为中心”向“以学为中心”加速演进。在这一进程中，如何提升教学资源的可及性、增强课堂互动的深度、满足学生个性化学习需求，成为教育高质量发展的核心命题。与此同时，人工智能技术的普及为教育创新提供了前所未有的支撑，推动教学工具从静态展示向智能交互跃升。

本项目立足于一线教学实际，面向教师备课负担重、学生参与度不足、语言类教学反馈滞后等现实痛点，构建一个智能语音辅助教学系统。该系统旨在通过自然、流畅、可交互的语音能力，将传统文本教案转化为富有表现力的听觉内容，支持有声课件自动生成、课文朗读与语言练习等典型场景，从而丰富教学形式、延伸学习时空、提升学习体验。

项目的实施不仅有助于减轻教师重复性工作负担，提高教学资源制作效率，更能为学生提供即时、可重复、可交互的学习支持，尤其在语言学习、自主预习与复习等环节发挥积极作用。长远来看，该系统有望成为智慧教育生态中的重要一环，推动优质教育资源的普惠共享，促进更加公平、高效、个性化的教育新形态发展。

二、 现状

  1. 多类型任务耦合在一起了，需要进行解耦，使用不同的接口、数据库来管理。
  2. 存在冗余的python服务，仅用于异步调用API，完全可以用java服务调用，且减少mq部署的资源消耗，LLM需要进行prompt优化。
  3. 需重新申请阿里云服务器进行部署开发。
  4. 阉割掉声音样本库这个页面，自定义样本成本、复杂度过高，统一使用预制音频。
  5. 功能不够完善且未充分测试验证。
  6. 需要进一步挖掘项目的技术亮点。

三、 设计方案

3.1 SoundMentor总体业务架构图

之前使用的阿里云服务器是为了方便团队协作开发，这里个人开发中间件直接在本机的docker上容器化部署即可。

3.3 各模块设计

3.3.1 有声ppt任务

流程：

  1. 用户上传ppt（上传接口），创建一个有声ppt任务（创建接口），创建1个task、n个task_detail,生成每页预览图上传并落库url，页面查询展示ppt的预览图，任务状态为创建。
  2. 用户执行讲解生成任务，任务状态改为讲解生成中并提交异步任务，异步任务解析每页内容并发请求LLM获取讲解落库，所有页完成后流传状态->讲解已生成，前端轮训获取状态和信息。
  3. 用户可以进行讲解的微调润色，进行讲解语音生成，任务状态改为语音生成中并提交异步任务，异步任务并发执行每页的讲解语音生成，请求api获取音频并上传获取url落库，所有页完成流转状态->语音已生成。
  4. 用户可以对各页语音进行编辑，上传自己的语音进行替代，进行最终的ppt生成，状态改为有声ppt生成中并提交异步任务，异步任务将每页的讲解音频嵌入到原ppt并生成终版文件，上传并落库url改状态为有声ppt已生成。
  5. 前端全程轮训获取实时状态。

需求接口：

  1. ppt任务创建接口
  2. 讲解生成接口
  3. 讲解编辑接口
  4. 讲解语音生成接口
  5. 讲解语音编辑接口
  6. 有声ppt生成接口
  7. 任务查询接口（根据用户id查询，任务id查询）

ai能力迁移：

  1. 原先执行算法任务（讲解生成，语音生成）是通过发送mq给python服务，然后消费python返回的结果消息进行落库。
  2. 现在调用ai接口迁移到java服务中，这里在用户接口内需要异步执行任务进行落库，三个单例线程池（讲解生成、讲解语音生成、有声ppt生成），用于异步执行三个步骤。
  3. 用户调用讲解生成接口，后台获取到任务的ppt_url,给线程池异步执行（解析ppt每页内容，并发请求LLM获取讲解进行落库，这里用CountDownLatch等所有页讲解落库后修改任务状态为讲解已生成）。
  4. 用户调用讲解语音生成接口，和‘3’类似，将返回的音频上传minio然后将url落库。

  2. 取消声音库训练等业务，简化声音库管理。

3.3.3 预设文本朗读

流程：

  1. 用户在输入框输入文字，并选择语音样本、语速等属性。
  2. 开始生成语音，调用接口创建一个任务，状态：创建，并提交一个异步任务给线程池进行文本转语音。
  3. 异步任务开始执行，将任务状态改成语音生成中，调用tts接口生成语音，将mp3文件上传minio并更新任务结果和状态（语音已生成）。
  4. 前端通过轮询来实时获取结果和状态，这里应该直接轮询这个用户的所有任务，前端需要能够看到历史任务的详情、状态、结果。
