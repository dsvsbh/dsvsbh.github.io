利用覆盖索引优化上千条回表操作为一次主键查询
在SoundMentor-基于ai语音合成教辅系统的项目里有个场景是需要根据语言和类型获取一个随机内容给用户练习发音，这里需要频繁根据语言和类型去获取随机内容 由于语言和类型需要频繁做条件查询，则向这两个字段建立联合索引 1create index language_type_index on language_content(language,type); 需要查询所有满足条件的记录并返回一个随机记录，如果是 select * 返回的话，由于联合索引是二级索引，叶子节点没有所有列的值，需要做回表查询，记录比较多需要回表查成千上万条记录影响性能 于是这里优化，只根据条件查id列表，由于叶子节点上有主键和索引列值，索引覆盖不需回表，拿到ids后再获取个随机id，走主键索引查单条记录 1234SELECT id FROM language_content WHERE language = #{language} AND type = #{type}
