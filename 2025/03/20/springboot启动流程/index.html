springboot启动流程

发表于2025-03-20|更新于2025-09-11|八股
|浏览量:

Spring Boot 的启动流程可以简化为以下几个主要步骤：

1. 初始化 SpringApplication

  • 加载 SpringApplication 实例，设置应用的主配置类（primarySources）。
  • 推断应用类型（如 Servlet、Reactive 等）。
  • 加载 ApplicationContextInitializer 和 ApplicationListener。

2. 准备环境（Environment）

  • 创建并配置应用的运行环境（Environment）。
  • 加载配置文件（如 application.properties 或 application.yml）以及命令行参数。

3. 创建应用上下文（ApplicationContext）

  • 根据应用类型（Servlet 或 Reactive）创建对应的 ApplicationContext。
    • 例如，Servlet 应用会创建 AnnotationConfigServletWebServerApplicationContext。

4. 准备应用上下文

  • 配置应用上下文的环境（Environment）。
  • 调用 ApplicationContextInitializer 初始化上下文。
  • 加载 Bean 定义（如通过 @ComponentScan 扫描的 Bean）。

5. 刷新应用上下文

  • 调用 ApplicationContext 的 refresh() 方法，这是 Spring Boot 启动的核心步骤。
    • 初始化 Bean 工厂。
    • 加载并注册 Bean 定义。
    • 执行 Bean 的实例化、依赖注入和初始化。
    • 启动内嵌的 Web 服务器（如 Tomcat）。

6. 执行 Runner

  • 调用所有 CommandLineRunner 和 ApplicationRunner 的 run 方法，执行自定义启动逻辑。

7. 启动完成

  • 应用启动完成，开始处理请求（如果是 Web 应用）。

总结

Spring Boot 的启动流程可以简化为以下几步：

  1. 初始化 SpringApplication。
  2. 准备环境。
  3. 创建应用上下文。
  4. 准备应用上下文。
  5. 刷新应用上下文。
  6. 执行 Runner。
  7. 启动完成。

这些步骤确保了 Spring Boot 应用的顺利启动和运行。
